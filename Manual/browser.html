The main LSD Browser has two purposes: to show the contents of one Object type (referred as “the Object pointed by the Browser”) and to serve as the control center for all LSD activities before and after a simulation run. The Object shown can be edited (e.g., add a new Variable, change its number of instances, set the initial data).

The LSD Browser is one window of the current LSD model program. The LSD Browser window is automatically open by LMM after a successful compilation of the current LSD model. Directly launching the LSD model program is also possible but may require setting up the system environment previously (according to the computer configuration).

The Browser window

The Browser window is formed by five elements: (1) the menu bar, (2) a shortcut tool bar, (3) the current Object information bar, (4) the descending Objects list box, and (5) the Variables and Parameters list box.

The menu bar contains all commands available in the Browser and is divided in five submenus (File, Model, Data, Run, Help). Each submenu in the menu bar is described in details in the next sections.

The tool bar offers shortcuts to the most frequently used commands, making their access simpler. The icons in each shortcut icon represent the respective command. A single mouse click on the shortcut executes the associated command. Hovering with the mouse over the icons shows the command name on the right part of the tool bar.

The Browser has three main information areas, described below:

·         Current Object information bar

·         Descending Objects list box

·         Variables and Parameters list box

Current Object information bar

The red label indicates the name of the Object pointed to by the Browser. Any operation specific to an Object will concern all the instances in the model of this type of Object.
Clicking on the red label is a shortcut to entry "Change Object Name" in menu Model.

The red label reports the name of the Object from which the currently shown Object descends. The label can be clicked to move the Browser to point to that Object, that is "up" in the hierarchy.

Descending Objects list box

Listbox containing the list of descendants from the Object shown in the Browser. The names of the descendants can be double-clicked and the Browser will move to show that descendant. A model in LSD is composed by a hierarchical structure of Objects: the whole model has a unique root, whose label is Root. Any Object can have any number of descendants, both intended in terms of different types of descendants and different numbers of instances for each Object type. The main Browser considers only Object types, without specifying the actual number of instances of each type. Therefore, browsing the model using the main Browser can be done by clicking on the descendants' names to go down the hierarchy and clicking on the label of the parent Object to go up the hierarchy.

Variables and Parameters list box

The listbox Variable contains all the Variables and the Parameters in the Object pointed by the Browser. The order in which the Variables and Parameters are listed depends on the order in which they have been inserted in the model, and it has no relevance for the simulation actual order of computation during a time step (for this see the LSD Simulation Manager help page).

After the label of the Variables and Parameters there is a code between parentheses:

·         (par.) means that the label corresponds to a Parameter;

·         (var. lag=X), where X is a non-negative integer, means that the label corresponds to a Variable. If n equals zero, it means that the Variable is never used in the model with lagged values. Positive n's mean that the corresponding Variable is used with a maximum lag equal to n.

Editing elements

Double-clicking on one element in the Variables and Parameters list opens the Change Element window (below).

The top part of the window indicates the label of the Variable or Parameter and the Object where it is contained. Double-click on the name of the Variable or Parameter to change it in another type of element (e.g., change the label, or turn a variable into a parameter).

Below the title there are three options (two for Parameters):

·         The field Save checked on means that the value of the Variable or Parameter must be saved in the result file for post-simulation analysis.

·         The field Run-time Plot means that the value of the Variable or Parameter must be plotted during the simulation run on the Run Time Plot. Note that this does not cause the automatic saving of the values in the result file for post-simulation analysis.

·         The field Debug (only for Variables) checked on means that the computation of any instance of the Variable causes the debugger to stop the simulation and show the state of the model (see LSD Debugger). The simulation will be interrupted immediately after the computation of the Variable.

The following buttons indicates to continue saving the options (OK), to continue without saving the modified options (Cancel) and the Help button for this page.

The parts below refer to the documentation of the Variable or Parameter. The "Description" is a text that the modeler can edit to indicate the meaning of the item.

The following two options, Observed and Initialized are used in the documentation of the model. If checked, the model will include this item in the list of elements containing relevant results and in the list of initializations. These lists are issued from the menu Run and placed in the header of the model report.

The four buttons below the Description (three for Parameters) have the following effects:

·         View code: show in a new window the actual code used to compute the Variable (not present for Parameters). See the manual page on the LSD equations (C++ codying style or the new LSD Macro codying style)

·         Auto Descr.: ask the system to provide automatic documentation for the Variable, which will replace any text possibly present. If the item is a Variable the system scans the equation's code looking for a commented part in the beginning of the equation. If found, this comment is copied in the Description. Moreover, both for Variables and for Parameters, the system also reports in which other equations the item is used.

·         Using Element: clicking this button creates a list of Variables whose equation makes use the element.

·         Elements Used: clicking on this item creates a new window with the list of items used in the equation for the element.

The last part of the window contains place to store comments on the initialization of the item (of course, it is present only for items with initial values: Parameters and lagged Variables). This comment is automatically filled when the user makes use of the Set All function in the Initial Values window.

The description part and the comments on the initial values are used in the model report in the cell concerning the item.

Change the properties of the element

WARNING: changing the properties of an element or its location in the Objects' structure can severely compromise the model. Use this option only to fix errors.

After having inserted a Variable or a Parameter it is possible to change the properties of the element by changing its label, transforming a Parameter in a Variable, a Variable in a Parameter, moving it to another Object or changing the maximum number of lags for a variable. If the model is running fine, the only reason for such a change is when the equations are changed, for example when you want to transform a Parameter in a Variable.

To change the nature of an element you need to:

1)    use the LSD Browser to show the object containing the element you want to change;

2)    on the element entry in the list of Variables and Parameters, you may OR double-click to show the options for the element and click the button Properties OR right-click on the element and choose Properties in the pop-up context menu;

3)    use the Properties dialog window (below) to edit the element properties as required.

The available operations are:

  • Name: edit the label of the element. To delete the element altogether assign an empty label, that is, delete all character in this box
  • Lags: change the maximum number of lagged values to be used in the computation of Equations (applicable only for Variables).
  • Type: the type of the element among:
    • Variable: turn the element into a variable.
    • Parameter: turn the element into a parameter.
    • Function: turn the element into a function.

Move the element to another Object

To change the Object in which an element is contained you need to:

1)    use the LSD Browser to show the object containing the element you want to move;

2)    on the element entry in the list of Variables and Parameters, you may OR double-click to show the options for the element and click the button Move OR right-click on the element and choose Move in the pop-up context menu;

3)    in the Move dialog window, select the Object to which the element should be moved to.

Delete the element from the model

To change the Object in which an element is contained you need to:

1)    use the LSD Browser to show the object containing the element you want to delete;

2)    on the element entry in the list of Variables and Parameters, you may OR double-click to show the options for the element and click the button Delete OR right-click on the element and choose Delete in the pop-up context menu;

3)    in the Confirmation dialog window, choose Yes to confirm the deletion.
